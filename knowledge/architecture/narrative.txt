llm-switch implements a two-part autonomous learning architecture as explicitly described in PRD Section 4.2 User Journeys, combining real-time intelligent model selection with offline self-learning to continuously improve routing decisions without requiring ongoing manual intervention from developers or operations teams. Developers integrate the system by redirecting API calls to llm-switch, benefiting from automatic model selection based on complexity, latency, and cost without code changes. Operations engineers deploy llm-switch in the Nomad cluster using Consul for service discovery and Vault for secret management, reducing operational overhead. The system provides Prometheus metrics at /metrics and integrates with Grafana dashboard llm-switch-overview for real-time visibility. Hardware-aware routing uses memory-priority strategy to assign workloads based on VRAM availability and queue depth, while Kubernetes Node Affinity constraints ensure optimal GPU scheduling. llm-switch supports OpenAI and Anthropic-compatible APIs for seamless integration with existing AI applications. Security boundaries define a trusted zone requiring mTLS for Consul and Vault, with frontier API access protected by TLS 1.3 termination. By routing requests to local models when feasible, llm-switch reduces frontier model API costs while maintaining response times under 500ms for 95% of requests. The offline self-learning system analyzes routing decisions overnight to refine orchestration thresholds, progressively increasing local model utilization and improving cost efficiency without manual intervention. It utilizes the AutoResearch loop to perform 5-minute training experiments that validate routing improvements before deployment.
